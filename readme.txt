Аминистратор: django_admin, пароль: adminpswd

Создан пользователь: user1, пароль: log_pswd

Создана группа 'log'. Пользователь user1 входит в данную группу.

При входе с гл. страницы user1 попадает в админскую часть - имеет доступ только к просмотру модели Logs. Сделан поиск по ip адресу, фильтрация по status_code, по выборке дат, по ip.

При входе администратора происходит переход на гл. стр. админки.

Если создать нового пользователя и не присвоить ему группу 'log' то при авторизации данного пользователя будет отказано.

API расположено по адресу http://127.0.0.1:8000/api/ (пример на локальном сервере) 

Для доступа к api также нужна авторизация. Логи могут просматривать только те пользователи, которые входят в группу 'log'. 
В апи сделана фильтрация по ip адресу и выборке дат. 

Модель Logs заполняется данными из скрипта parse_to_db, который лежит в main/scripts.

Данный скрипт заполняет данные бд из файла access.log. Для примера я положил пример access.log в туже папку(scripts), что и скрипт parse_to_db.

Скрипт можно запустить вручную. Активируем вирт окружение проекта.
Затем запускаем скрипт командой ./manage.py runscript parse_to_db.

Автоматичеки запуск скрипта настраивается в cron. 
Настриваем cron расписание командой crontab -e
Пишем запуск скрипта на ежедневный запуск к примеру в 8:00 утра
0 8 * * * /home/project/django_task/env/bin/python /home/project/django_task/manage.py runscript parse_to_db

Путь /home/project/django_task/env/bin/python - это пример пути до имперпритатора python вирт окружения проекта 

При размещении автоматического запуска в скрипте parse_to_db нужно изменить путь до файла access.log. К примеру в линуксе access.log Nginx находится по пути /var/log/nginx/access.log
